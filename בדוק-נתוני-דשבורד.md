# 🔍 בדיקת נתוני דשבורד

## הבעיה
אתה אומר שבדשבורד אין נתונים אמיתיים.

## מה לבדוק

### 1. פתח את הדשבורד
```
http://localhost:5174/dashboard
```

### 2. פתח Console (F12)
בדוק את הלוגים:
```
📊 Dashboard - Pages Count: ?
📊 Dashboard - Subscription: ?
📊 Dashboard - User ID: ?
📄 First Page - Title: ?
📄 First Page - Slug: ?
📄 First Page - Type: ?
```

### 3. בדוק מה מוצג בכרטיסי הדפים

האם אתה רואה:
- ✅ תמונת רקע של הדף?
- ✅ כותרת הדף?
- ✅ תיאור הדף?
- ✅ סוג הדף (חנות/אירוע/שירות)?
- ✅ תאריך יצירה?
- ✅ סטטוס מנוי?

## מה אמור להיות מוצג

בכל כרטיס דף צריך להיות:
1. **תמונת רקע** - מ-`page.metadata.headerImage`
2. **כותרת** - מ-`page.title`
3. **תיאור** - מ-`page.description`
4. **סוג דף** - מ-`page.pageType`
5. **תאריך** - מ-`page.createdAt`
6. **סטטוס מנוי** - מ-`page.subscriptionStatus`

## אם אין נתונים

### בעיה 1: הדפים לא נטענים
```javascript
// בדוק ב-Console:
📊 Dashboard - Pages Count: 0
❌ No pages found!
```

**פתרון**: בדוק אם המשתמש מחובר ויש לו דפים ב-Strapi.

### בעיה 2: הנתונים חסרים
```javascript
// בדוק ב-Console:
📄 First Page - Title: MISSING TITLE
📄 First Page - Slug: MISSING SLUG
```

**פתרון**: הנתונים לא מגיעים מהשרת. צריך לבדוק את ה-API.

### בעיה 3: התמונות לא מוצגות
אם אתה רואה רק אייקון סגול במקום תמונה:
- התמונה לא קיימת ב-`page.metadata.headerImage`
- או שה-URL לא תקין

## איך לתקן

### תיקון 1: בדוק את הנתונים ב-Strapi
```bash
node check-hero-image-in-page.js
```

צריך לראות:
```
✅ Hero image EXISTS in Strapi!
🔗 Image URL: http://localhost:1337/uploads/...
```

### תיקון 2: בדוק את ה-API של הדשבורד
פתח:
```
http://localhost:5174/dashboard?userId=YOUR_USER_ID
```

ובדוק ב-Network tab מה מגיע מהשרת.

### תיקון 3: רענן את הדף
לחץ Ctrl+Shift+R (Windows) או Cmd+Shift+R (Mac) לרענון מלא.

## מה אני רואה עכשיו?

תאר לי בדיוק מה אתה רואה בדשבורד:
1. כמה דפים מוצגים?
2. האם יש תמונות?
3. האם יש כותרות?
4. האם יש כפתורים?
5. מה הלוגים ב-Console?

אחרי שתבדוק - ספר לי מה אתה רואה ואני אתקן! 🎯
