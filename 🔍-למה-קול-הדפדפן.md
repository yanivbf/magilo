# 🔍 למה אתה שומע את קול הדפדפן ולא Google TTS?

## 🎯 הבעיה:

אתה שומע את **קול הדפדפן** (Browser TTS) במקום **קול Google TTS** הרובוטי.

זה קורה כי הקוד **נופל ל-fallback** - כלומר, הבקשה ל-Google TTS נכשלת, אז הוא משתמש בקול הדפדפן.

---

## 🔍 איך לבדוק מה השגיאה:

### שלב 1: פתח Console
```
לחץ F12 בדפדפן
↓
לחץ על Tab "Console"
```

### שלב 2: נקה את הקונסול
```
לחץ על האייקון 🚫 (Clear console)
```

### שלב 3: דבר עם סתיו
```
כתוב משהו לסתיו
↓
שלח את ההודעה
↓
חכה לתשובה
```

### שלב 4: חפש שגיאות
```
חפש בקונסול:
❌ "Google TTS error"
❌ "Failed to fetch"
❌ "403" או "401" (שגיאות הרשאה)
❌ "quota" (חריגה ממכסה)
```

---

## 🎯 שגיאות אפשריות:

### 1️⃣ שגיאת CORS
```
❌ Access to fetch at 'https://texttospeech.googleapis.com...' 
   from origin 'http://localhost:5174' has been blocked by CORS policy
```

**פתרון**: צריך להשתמש ב-API endpoint דרך השרת שלנו

### 2️⃣ שגיאת API Key
```
❌ API key not valid. Please pass a valid API key.
```

**פתרון**: צריך API Key חדש מ-Google Cloud

### 3️⃣ חריגה ממכסה
```
❌ Quota exceeded for quota metric 'CharacterCount'
```

**פתרון**: צריך להגדיל את המכסה או להמתין

### 4️⃣ שגיאת רשת
```
❌ Failed to fetch
```

**פתרון**: בדוק חיבור לאינטרנט

---

## 📋 מה לעשות עכשיו:

1. **פתח Console** (F12)
2. **נקה** את הקונסול
3. **דבר עם סתיו**
4. **העתק** את השגיאה שאתה רואה
5. **שלח לי** את השגיאה

---

## 🔧 פתרונות זמניים:

### אם זה שגיאת CORS:
נצטרך להשתמש ב-API endpoint דרך השרת:
```
/api/tts → קורא ל-Google TTS מהשרת
```

### אם זה שגיאת API Key:
נצטרך API Key חדש מ-Google Cloud Console

### אם זה חריגה ממכסה:
נצטרך להמתין או להגדיל מכסה

---

## ✅ איך לדעת שזה עובד:

כשזה עובד, תראה בקונסול:
```
🤖 Calling Google TTS API directly...
✅ Received audio from Google TTS
✅ Speech finished
```

ותשמע קול **רובוטי ומהיר** במקום קול הדפדפן הרגיל.
