# 🧪 בדיקה מהירה - למה המקטעים לא עובדים

## 🔍 הבעיה
אתה רואה HTML גולמי במקום מקטעים מעוצבים.

## ✅ מה תיקנו
1. ✅ `pages/[slug]/+page.svelte` - משתמש ב-PageRenderer
2. ✅ `PageRenderer.svelte` - מציג מקטעים רק אם יש תוכן
3. ✅ `+page.server.js` - מחלץ נתונים מ-metadata

## 🎯 למה זה עדיין לא עובד?

### הסיבה: הדף הישן לא מכיל נתונים מובנים

הדף הישן שלך ב-Strapi נראה כך:
```json
{
  "htmlContent": "<div>...</div>",
  "metadata": {},
  "gallery": null,
  "testimonials": null,
  "faq": null
}
```

אבל PageRenderer מצפה ל:
```json
{
  "gallery": ["url1", "url2"],
  "testimonials": [{...}],
  "faq": [{...}]
}
```

## 🚀 פתרון מהיר - צור דף חדש

1. **לך ל-Page Creator**: `http://localhost:5173/page-creator`

2. **צור דף חדש** עם המקטעים האופציונליים:
   - ✅ סמן "גלריית תמונות"
   - ✅ סמן "המלצות"
   - ✅ סמן "שאלות ותשובות"
   - ✅ סמן "אודות"

3. **שמור את הדף**

4. **פתח את הדף החדש** - תראה את המקטעים המעוצבים!

## 🔄 להמיר דפים ישנים

אם יש לך דפים ישנים שאתה רוצה להמיר:

### אופציה 1: ערוך ידנית
1. פתח את דף הניהול של הדף הישן
2. הוסף תוכן למקטעים (גלריה, המלצות, FAQ)
3. שמור
4. הדף יישמר עם המבנה החדש

### אופציה 2: הרץ סקריפט המרה
```bash
# הגדר את ה-API token
set STRAPI_API_TOKEN=your-token-here

# הרץ את הסקריפט
node scripts/migrate-pages-to-structured.js
```

## 🎨 איך לבדוק שזה עובד

1. צור דף חדש עם מקטעים אופציונליים
2. פתח את הדף: `http://localhost:5173/pages/[slug]`
3. תראה:
   - ✅ גלריה מעוצבת (לא HTML)
   - ✅ המלצות מעוצבות (לא HTML)
   - ✅ FAQ מעוצב (לא HTML)
   - ✅ אודות מעוצב (לא HTML)

## 💡 למה זה קורה?

המערכת הישנה שמרה הכל כ-HTML גולמי.
המערכת החדשה שומרת נתונים מובנים + מציגה אותם בעיצוב Svelte.

דפים ישנים = HTML גולמי בלבד
דפים חדשים = נתונים מובנים + עיצוב Svelte

## ✅ מה לעשות עכשיו?

1. **צור דף חדש** לבדיקה
2. **ודא שהמקטעים מוצגים בעיצוב מלא**
3. **אם עובד** - המערכת תקינה! רק צריך להמיר דפים ישנים
4. **אם לא עובד** - תראה לי את הקונסול ונבדוק ביחד
