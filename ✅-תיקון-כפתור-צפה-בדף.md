# ✅ תיקון כפתור "צפה בדף" - הושלם

## מה תוקן?

כפתור "צפה" בדשבורד עכשיו פותח את הדף בתצוגה ציבורית נקייה, **ללא סרגל עריכה**.

---

## השינויים שבוצעו

### 1. עדכון פונקציית `viewPage()` בדשבורד
**קובץ:** `new-app/src/routes/dashboard/+page.svelte`

```javascript
function viewPage(page) {
    // Open page in new tab - use /view/ for public view
    const slug = page.slug || page.fileName || page.documentId || page.id;
    console.log('🔍 Opening page:', { slug, page });
    if (!slug) {
        alert('שגיאה: לא נמצא slug לדף זה');
        return;
    }
    // Use /view/ route for public viewing (no edit toolbar)
    window.open(`/view/${slug}`, '_blank');
}
```

**מה השתנה:**
- הכפתור עכשיו פותח את `/view/${slug}` במקום `/pages/${slug}`
- זה מבטיח תצוגה ציבורית נקייה ללא כלי עריכה

---

### 2. הפניה אוטומטית מ-`/pages/[slug]` ל-`/view/[slug]`
**קובץ:** `new-app/src/routes/pages/[slug]/+page.server.js`

```javascript
export async function load({ params, locals }) {
    const { slug } = params;
    
    // Redirect to /view/[slug] for public viewing
    throw redirect(302, `/view/${slug}`);
}
```

**מה השתנה:**
- כל גישה ל-`/pages/[slug]` מועברת אוטומטית ל-`/view/[slug]`
- ההפניה מתבצעת בצד השרת (לפני טעינת הדף)
- אין יותר הודעת "מעביר לדף..." - ההפניה מיידית

---

## איך זה עובד עכשיו?

### מסלולי הדפים:

1. **`/view/[slug]`** - תצוגה ציבורית
   - ללא סרגל עריכה
   - ללא כפתורים של עריכה
   - רק התוכן הציבורי של הדף
   - **זה מה שנפתח כשלוחצים "צפה"**

2. **`/pages/[slug]`** - מועבר אוטומטית ל-`/view/[slug]`
   - הפניה מיידית בצד השרת
   - שומר על תאימות לאחור

3. **`/edit/[slug]`** או **Page Creator** - מצב עריכה
   - עם כלי עריכה מלאים
   - **זה מה שנפתח כשלוחצים "ערוך"**

## בדיקה

### לחץ על "צפה" בדשבורד:
1. פתח את הדשבורד: http://localhost:5173/dashboard
2. לחץ על כפתור "צפה" בכרטיס של דף
3. **תוצאה:** הדף נפתח בחלון חדש ב-`/view/[slug]`
4. **אין סרגל עריכה** - רק התוכן הציבורי

### לחץ על "ערוך" בדשבורד:
1. לחץ על כפתור "ערוך" בכרטיס של דף
2. **תוצאה:** נפתח Page Creator במצב עריכה
