# בדיקת מערכת מובנית חדשה

## מה עשינו עד כה

✅ יצרנו API endpoint חדש: `/api/create-structured-page`
✅ עדכנו את page-creator לקרוא ל-API החדש
✅ הוספנו פונקציה `createProduct` ב-strapi.js
✅ עדכנו את `getPageBySlug` לטעון products
✅ עדכנו את +page.server.js לטעון products מה-relation
✅ עדכנו את PageRenderer להעביר products ל-ProductsGallerySection

## איך לבדוק

### 1. הפעל את Strapi
```bash
cd strapi-backend
npm run develop
```

### 2. הפעל את האפליקציה
```bash
cd new-app
npm run dev
```

### 3. צור דף חנות חדש
1. פתח http://localhost:5173/page-creator
2. בחר "חנות מקוונת"
3. מלא את הפרטים
4. בחר מקטעים אופציונליים (Gallery, FAQ, וכו')
5. לחץ "צור דף"

### 4. בדוק שהדף נוצר נכון
- המוצרים מופיעים בגלריה?
- המקטעים האופציונליים מופיעים?
- הכל נראה טוב?

## מה צפוי לקרות

1. ה-API יוצר דף ב-Strapi **ללא HTML**
2. ה-API יוצר sections (about, gallery, faq, testimonials, products)
3. ה-API יוצר products בטבלה נפרדת
4. PageRenderer טוען את הנתונים המובנים
5. ProductsGallerySection מציג את המוצרים

## אם משהו לא עובד

בדוק את הקונסול:
- האם יש שגיאות ב-Strapi?
- האם המוצרים נשמרים ב-database?
- האם ה-sections נוצרים?
- האם PageRenderer מקבל את הנתונים?

## הצעד הבא

אחרי שנוודא שהכל עובד, נוכל:
1. למחוק את ה-API הישן (`create-page-with-template`)
2. למחוק את הקוד שמטפל ב-HTML
3. לבנות ממשק עריכה למקטעים
